Welcome
Workshop Overview

We're going to build a real, working application together. We will create a personal note taking tool using an agentic AI coding tool. We'll do this through prompting alone, without writing any code ourselves.

Core Concepts 1/4
The Mental Model Shift

We won't be writing any code today. Instead, we will direct a very fast, very literal software engineer who happens to know every programming language ever written.

Core Concepts 2/4
The Development Cycle

Every feature you build today follows the same loop. When you get stuck, two branches help you get unstuck.

Core Concepts 3/4
Vibe Coding Challenges

Vibe coding introduces specific frictions: ambiguity, context drift, and nondeterminism, that can disrupt your project. The right response to all of these challenges is to stay calm, diagnose what the prompt got wrong, and iterate.

Personal Customization
Now it's your turn
Build something on your own
  • I'll circulate — raise your hand if you get stuck
  • Choose an option below and paste the prompt directly into Antigravity
  • Test it thoroughly, click around like a real user would
  • If something breaks, tell the model what you expected vs. what happened
  • Stuck badly? Type "revert to previous version" to undo the last change
  • Try as many as you like, or create your own
Option 1 — Dark mode
Add a dark mode toggle button. Clicking it switches the app to a dark color scheme. Clicking it again switches back.
Option 2 — Sort order
Add a sort control to the right side of the unified search toolbar, separated from the category filter by a vertical divider. It should be a button labeled 'Sort: Newest' by default. Clicking it toggles the notes list between newest-first and oldest-first order, and updates the button label to reflect the current sort direction.
Option 3 — Edit notes
Make each note card clickable. Clicking anywhere on a note populates the form with that note's content and changes the Save button label to 'Update Note'. Saving updates the existing note in place. Add a small subtle 'cancel' text link next to the Update Note button that clears the form and returns it to new-note mode.
Option 4 — Delete notes
Add a small round delete button in the top-right corner of each note card, no larger than 20px, showing only an ✕ symbol. It should be visible by default without changing any other styling on the card. Clicking it asks the user to confirm before permanently removing the note.
Option 5 — Export Notes
Add a fixed footer bar spanning the full width of both columns, matching the current app design. Place a right-aligned Export CSV button inside it that downloads all notes as a CSV file with columns for title, date, category, and body.
Option 6 — Your Own Idea
Describe your feature, what triggers it, what changes in the UI, and any edge cases. Avoid vague words — say exactly what you want to see. If the result isn't right, prompt again to fix, or type "Revert to previous version" to undo the last update.
Section 5 · Working Like a Pro
Persistent Instructions
Rules files
  • Encode decisions upfront so you don't repeat yourself in every prompt
  • These are project-specific rules, but most tools also support global defaults
  • Rules file conventions are evolving — research before use
Antigravity: ~/<your-project>/.agent/rules/project.mdc Claude Code: ~/.claude/CLAUDE.md
# Persona (How To Work) ## Interaction Style - Be Concise: No conversational filler like "I've updated the file for you." - Micro-Steps: Propose one logical change at a time. - Clarification: If a prompt is ambiguous, ask ONE clarifying question before coding. ## Technical Baseline - Environment: macOS + Chrome. - Simplicity First: Always default to single-file Vanilla HTML/CSS/JS unless a build step is explicitly requested. ## Context Management - Handoff: If the conversation gets long, provide a "State Summary": - What is working. - What is currently "broken" or half-finished. - What the very next step should be. --- # Rules: (What We're Building) ## Architecture & Layout - Two-Column Layout: Note entry form (left), notes list (right). - Unified Toolbar: Full-width top bar. - Left side: Search input. - Right side: Category filter dropdown. - Detail: Use vertical dividers between items. ## Visual Identity - Theme: Minimalist. White background, neutral gray accents. - UI Specs: 8px rounded corners, subtle card shadows, readable sans-serif. ## Data & Features - Fields: Title, Body, Date (auto), and Category. - Persistence: All data stays in localStorage. - Logic: - Real-time search (matches title/body). - Dynamic filtering (category list grows as new categories are typed). - Single-page state management (no reloads).
